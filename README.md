# Metallprofil Price Tracker

Локальный MVP для отслеживания прайсов `Металл Профиль`.

На текущем этапе автоматизирована только группа:

- `Сэндвич-панели`

## Что делает проект

Скрипт берет PDF-прайсы из папки `input/`, извлекает из них данные по сэндвич-панелям и собирает:

- `data/sandwich-panels.json` — структурированные данные для дальнейшей обработки
- `site/index.html` — готовую статическую страницу с выбором даты прайса

Сейчас проект умеет:

- показывать реальные даты прайсов из папки `input/`
- сравнивать актуальный прайс с предыдущим
- строить таблицы по классам 1 и 2
- показывать фасонные изделия для ТСП
- показывать стандартные цвета и минимальный заказ

## Что делать при появлении нового прайса

1. Положить новый PDF в папку `input/`.
2. Убедиться, что в имени файла есть дата, например `03.04.2026`.
3. Запустить:

```powershell
python scripts/update_sandwich_panels.py
```

4. Проверить результат в:

- `data/sandwich-panels.json`
- `site/index.html`

5. Открыть страницу и проверить:

- появилась ли новая дата в переключателе
- корректно ли отображаются таблицы класса 1 и класса 2
- совпадает ли дата действия прайса с тем, что написано в PDF
- не сломался ли блок фасонных изделий и стандартных цветов

6. Если все корректно, закоммитить и запушить изменения.

## Быстрый локальный запуск через BAT

Парсинг выполняется локально на вашем компьютере.

Для автоматического сканирования папки и обновления проекта используйте:

```bat
scan_and_update_input.bat
```

По умолчанию скрипт сканирует `%USERPROFILE%\Downloads`, копирует новые PDF в `input/` и запускает:

```powershell
python scripts/update_sandwich_panels.py
```

Можно передать свою папку-источник:

```bat
scan_and_update_input.bat "C:\Папка\С\Прайсами"
```

## Как определяется порядок прайсов

Скрипт сортирует PDF по дате, которую пытается извлечь из имени файла.

Поддерживаются варианты:

- `dd.mm.yyyy`
- `dd.mm.yy`
- `dd.mm`

Если года в имени нет, он берется из даты изменения файла.

## Что именно парсится из PDF

Из каждого PDF проект извлекает:

- страницу с прайсом на сэндвич-панели
- страницу со стандартными цветами
- дату действия прайса
- цены по толщине
- минимальный заказ
- данные для фасонных изделий

## Как парсится страница 28

Да, логика парсинга страницы 28 зашита в коде программно.

Сейчас в [scripts/update_sandwich_panels.py](C:\Металлпрофиль\repo-git\scripts\update_sandwich_panels.py) это устроено так:

- функция `parse_sandwich_page()` ищет страницу не по номеру `28`, а по ключевым словам
- для поиска используются признаки:
  - заголовок `ТРЕХСЛОЙНЫЕ СЭНДВИЧ-ПАНЕЛИ "МЕТАЛЛ ПРОФИЛЬ"`
  - наличие `МП ТСП-Z`
  - наличие `ДОПОЛНИТЕЛЬНЫЕ ОПЦИИ`
- после нахождения страницы скрипт вытаскивает:
  - таблицу цен по `МП ТСП-Z`, `МП ТСП-S`, `МП ТСП-К`
  - деление на `Класс 1` и `Класс 2`
  - дату действия прайса
  - упоминания фасонных изделий для ТСП

Для фасонных изделий на сайте сейчас используется формула со страницы 28:

- `цена плоского листа × 1,9`

Базовые цены для этой формулы теперь тоже берутся автоматически из PDF:

- из строки `1 Плоский лист ТУ` (прайс с полимерным покрытием) для `Полиэстер 25 мкм`
- из строки `1 Плоский лист **` (прайс №2, оцинкованные) для `Оцинковка`

После этого скрипт сам рассчитывает итоговые значения для фасонных изделий и записывает их в `accessories_prices`.

## Ограничения текущей версии

- проект завязан на текущую структуру PDF
- если в новом прайсе изменятся заголовки, подписи или верстка страницы, парсер может перестать находить нужные блоки
- страница с ценами ищется по ключевым словам, а не по жесткому номеру
- автоматизация покрывает только группу `Сэндвич-панели`
- парсинг выполняется локально, GitHub Pages только публикует готовый `site/index.html`

## Запуск

```powershell
python scripts/update_sandwich_panels.py
```

## Структура

- `input/` — входные PDF
- `data/` — распарсенные данные
- `scripts/` — парсер и генерация
- `site/` — готовая страница для GitHub Pages

## Если новый PDF не парсится

Сначала проверить:

- сохранились ли в PDF нужные заголовки
- есть ли на странице те же маркеры, что ищет скрипт
- не изменилась ли структура блока цен
- не изменилась ли страница со стандартными цветами

Главные точки для правки:

- `parse_sandwich_page()`
- `parse_colors_page()`
- `find_page_index_by_terms()`
